     4→## Overview
     5→
     6→Clay.com is the most playful B2B SaaS interface in the GTM-data category. The base atmosphere is **cream-tinted white canvas** (`{colors.canvas}` — #fffaf0) holding dark-navy ink type and **3D-rendered claymation illustrations** (mountains, mascot characters, peach/ochre/lavender landscapes) as the dominant brand voltage. Where most data-platform brands play it cool with grids and gradients, Clay leans hard into hand-crafted-looking 3D illustrations and saturated single-color feature cards.
     7→
     8→Type voice runs **Plain Black** (or substituted with Inter weight 500-600) — a custom rounded display face used at very large sizes (72px hero) with negative letter-spacing. Body type uses Inter at standard weights. The display weight stays at 500, never bolder — the rounded character of the typeface gives it warmth without needing weight.
     9→
    10→Component voltage comes from **saturated single-color feature cards** in a 6-color palette: hot pink, deep teal, lavender, peach, ochre, and cream-card. Each card shows product UI fragments at small scale — Claygent agent runs, sequencer flows, CRM enrichment outputs. The colored card IS the primary visual element on every long-scroll page.
    11→
    12→**Key Characteristics:**
    13→- Cream-tinted white canvas (`{colors.canvas}` — #fffaf0). The warmth differentiates Clay from cool-gray competitor sites.
    14→- Dark navy/black primary CTAs (`{colors.primary}` — #0a0a0a). Buttons rounded `{rounded.md}` (12px) — friendly modern but not pill.
    15→- 6-color saturated feature card palette: `{colors.brand-pink}`, `{colors.brand-teal}`, `{colors.brand-lavender}`, `{colors.brand-peach}`, `{colors.brand-ochre}`, `{colors.surface-card}` (cream).
    16→- 3D claymation illustrations (mountains, characters, abstract shapes) as full-bleed hero artifacts — the brand's most-recognized visual element.
    17→- Custom rounded Plain Black display typeface at 500 weight with -1 to -2.5px letter-spacing on display sizes.
    18→- Border radius is generous: `{rounded.md}` (12px) for buttons + inputs, `{rounded.lg}` (16px) for content cards, `{rounded.xl}` (24px) for feature cards. The bigger radius matches the rounded display type's character.
    19→- Product UI fragments embedded inside colored cards at small scale — agent run logs, sequencer flows, enrichment results.
    20→- Section rhythm `{spacing.section}` (96px) between major bands.

    57→## Typography
    58→
    59→### Font Family
    60→The system runs **Plain Black** (a custom rounded display face) for headlines and **Inter** for body, navigation, and UI. Plain Black at weight 500 with negative letter-spacing handles every display headline; Inter handles the rest. The fallback stack walks `Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` for both.

    80→### Principles
    81→Plain Black at weight 500 + negative letter-spacing IS the brand voice. Going to weight 700 reads as bombastic; the rounded character of the typeface adds warmth that bolder weight would flatten.
    82→
    83→The body-vs-display split is functional: Plain Black for Plain Black moments (headlines), Inter for everything else (running text, UI, buttons). Mixing them is a system violation.
    84→
    85→### Note on Font Substitutes
    86→If Plain Black is unavailable, **Inter** at weight 500 with -0.05em letter-spacing is a usable approximation. **Söhne Breit** at weight Buch is an alternative if licensed. **Recoleta** at weight 500 carries similar rounded-display warmth.

   191→## Do's and Don'ts
   192→
   193→### Do
   194→- Anchor every page on the cream canvas (`{colors.canvas}` — #fffaf0). The warm tint differentiates Clay from cool-gray data sites.
   195→- Use 3D claymation illustrations as hero artifacts. Hand-crafted 3D characters and mountains ARE the brand.
   196→- Cycle saturated feature cards across the page — pink → teal → lavender → peach → ochre → cream. Repeating the same color twice in a row reads as off-rhythm.
   197→- Use Plain Black at weight 500 with negative letter-spacing on every display headline.
   198→- Show product UI fragments inside saturated feature cards. The brand voltage is product-driven, not abstract.
   199→- Use cream footer (NOT dark). Clay deliberately closes pages with warm cream rather than the standard dark-footer SaaS template.
   200→- Anchor every band with `{spacing.section}` (96px) vertical rhythm.
   201→
   202→### Don't
   203→- Don't use cool grays for canvas. The cream tint is non-negotiable.
   204→- Don't use a 7th brand-color card. The 6-color palette is saturated enough.
   205→- Don't bold display weight beyond 500. Plain Black at 700 reads as bombastic.
   206→- Don't repeat the same brand-color card twice in a row.
   207→- Don't replace claymation illustrations with flat vector art. The hand-crafted 3D character IS the brand voice.
   208→- Don't use a dark footer. The cream footer is part of the system's warm-throughout pacing.
   209→- Don't add hover state styling beyond what the system already encodes.

   326→Reicon provides a comprehensive set of SVG icons designed on a strict 24×24 pixel grid. The library is built for high-performance web applications, offering complete tree-shakeability, zero external dependencies, and optimized wrappers for multiple frameworks.
   327→All icons are maintained in two weights:
   328→•       Outline: 1.5px stroke-width with consistent corner radiuses.
   329→•       Filled: Solid path structures designed for active states or highlighted UI elements.

   410→# slot-text
   411→
   412→> Dependency-free text roll animation for tiny, tactile UI labels. Pure CSS transforms, no runtime dependencies. Browser-only DOM utility. Works best on short labels, buttons, counters, and command text.
   413→
   414→## Install
   415→
   416→```bash
   417→npm install slot-text
   418→```
   419→
   420→Import the CSS once before using the animation: `import "slot-text/style.css";`
   421→
   422→## Usage: Vanilla
   423→
   424→```ts
   425→import "slot-text/style.css";
   426→import { slotText, chromatic } from "slot-text";
   427→
   428→const label = slotText(document.querySelector("#copy"), "Copy");
   429→label.set("Copied", { direction: "up", color: chromatic() });
   430→label.set("Copy", { direction: "down" });
   431→label.destroy();
   432→```

   512→## API
   513→
   514→- `slotText(element, text, options?)` — vanilla controller with `.set(text, options?)`, `.flash(text, options?)`, and `.destroy()`
   515→- `SlotText` component from `slot-text/react` and `slot-text/vue` (optional peer deps; plain JS users do not need them)
   516→- `slotText` directive from `slot-text/solid` and `slot-text/svelte` (optional peer deps)
   517→- Low-level helpers: `buildSlotText`, `animateSlotText`, `chromatic` from `slot-text`

   519→## Options
   520→
   521→```ts
   522→type SlotOptions = {
   523→  direction?: "up" | "down";       // default "down"
   524→  stagger?: number;                // default 45
   525→  duration?: number;               // default 300
   526→  exitOffset?: number;             // default 50
   527→  easing?: string;                 // default "cubic-bezier(0.34, 1.56, 0.64, 1)"
   528→  bounce?: number;                 // default 0.6
   529→  color?: string | ((index: number, total: number) => string);
   530→  colorFade?: number;              // default 280
   531→  skipUnchanged?: boolean;         // default true
   532→  interrupt?: boolean;             // default true
   533→};
   534→```
   535→PARA LOS CARROUSELES  Y LA GALERIA USA BLOSOM Blossom Carousel
   536→  
   537→
   538→Blossom Carousel is the first carousel library that builds on native browser scrolling instead of replacing it. It provides a small drag enhancement for pointer devices while maintaining the core interaction model, performance, and accessibility of a real scroll container.
   539→
   540→🥇 Native scrolling: Full performance and accessibility.
   541→🚀 Dragging: Custom physics-based dragging for all pointer types.
   542→✨ No abstraction: use all native web API's.
   543→💡 Configure with CSS: Works with native scroll-snap, position sticky and scroll-driven animations and more.
   544→🪶 0kb on touch devices: Blossom only loads when a fine pointer device is detected.
   545→🧱 Framework ready: Components for React, Vue, Svelte and Web Components.
   546→🚧 Experimental cyclical scrolling: Allows the carousel to infinitely loop.

   639→Navigation controls
   640→Place previous, next, and dot controls outside the carousel with <BlossomPrev>, <BlossomNext>, and <BlossomDots>. Link them to the carousel with an id on <BlossomCarousel> and a matching for prop on each control.
   641→
   642→
   643→import {
   644→  BlossomCarousel,
   645→  BlossomPrev,
   646→  BlossomNext,
   647→  BlossomDots,
   648→} from "@blossom-carousel/react";
   649→import "@blossom-carousel/react/style.css";
   650→
   651→export function Gallery() {
   652→  return (
   653→    <>
   654→      <BlossomCarousel id="gallery" className="carousel">
   655→        {Array.from({ length: 12 }, (_, index) => (
   656→          <div key={index} data-blossom-slide className="slide">
   657→            Slide {index + 1}
   658→          </div>
   659→        ))}
   660→      </BlossomCarousel>
   661→
   662→      <div className="controls">
   663→        <BlossomPrev for="gallery" />
   664→        <BlossomDots for="gallery" />
   665→        <BlossomNext for="gallery" />
   666→      </div>
   667→    </>
   668→  );

   692→Overscroll API
   693→Tap into Blossom's drag engine's overscroll behavior to create your own style.
   694→
   695→
   696→import { useRef } from "react";
   697→import { BlossomCarousel } from "@blossom-carousel/react";
   698→
   699→export function App() {
   700→  const blossomCarousel = useRef<HTMLElement | null>(null);
   701→
   702→  function onOverscroll(event: CustomEvent<{ left: number }>) {
   703→    // prevent and overwrite Blossom's default rubberbanding effect
   704→    event.preventDefault();
   705→
   706→    const overScroll = event.detail.left;
   707→
   708→    Array.from(blossomCarousel.current?.children ?? []).forEach((slide) => {
   709→      (slide as HTMLElement).style.transform = `scale(${1 - overScroll * 0.1})`;
   710→    });
   711→  }
   712→
   713→  return (
   714→    <BlossomCarousel
   715→      ref={blossomCarousel}
   716→      onOverscroll={(event) => {
   717→        onOverscroll(event as CustomEvent<{ left: number }>);
   718→      }}
   719→    >
   720→      {Array.from({ length: 12 }, (_, index) => (
   721→        <div key={index}>Slide {index + 1}</div>
   722→      ))}
   723→    </BlossomCarousel>
   724→  );
   725→}
